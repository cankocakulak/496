Snake Game - Project Report

1. Application Description

This project is an implementation of the classic Snake game as a single HTML page web application with JavaScript. The Snake game is a classic arcade game where players control a snake that moves around a grid, eating food to grow longer while avoiding collisions with walls and itself. This implementation features modern design, responsive controls, and additional gameplay features to enhance the user experience.

The game offers two difficulty levels:

  • Easy: Moderate speed with 2 points awarded per food item
  • Hard: Faster speed with obstacles, 3 points per food item, and increasing challenge as obstacles appear when food is consumed

The game is designed to be intuitive, responsive, and accessible for users of various skill levels and devices. Special attention has been paid to applying HCI (Human-Computer Interaction) principles to create an engaging and user-friendly experience.

2. Design and Implementation

The Snake game was designed with a focus on simplicity and responsiveness. The key design decisions and implementation approaches are:

Design Approach

  • Canvas-based rendering: The game uses HTML5 Canvas for smooth graphics rendering
  • Responsive design: Adapts to different screen sizes with CSS media queries
  • Minimalist UI: Clean, distraction-free interface with important information clearly visible
  • Color psychology: Used green for the snake (representing growth), red for food (representing rewards), and purple for obstacles (representing danger)
  • Feedback mechanisms: Clear visual and interactive feedback for user actions

Implementation Details

  • Game loop: Uses setTimeout for game tick control with variable speeds based on difficulty
  • State management: Tracks game state (running, paused, game over) with appropriate UI updates
  • Collision detection: Checks for collisions with walls, snake body, and obstacles
  • Touch controls: Implemented swipe gestures for mobile users
  • Local storage: Persists high score between sessions using browser's localStorage API
  • Dynamic obstacle generation: Intelligent placement algorithm for obstacles in hard mode

3. Libraries and Tools Used

The Snake game was developed as a standalone web application using vanilla HTML, CSS, and JavaScript without external libraries. This approach keeps the application lightweight and dependency-free.

Technology/Tool Purpose
HTML5 Structure of the web page
CSS3 Styling, responsive design, and visual effects
JavaScript (ES6+) Game logic, event handling, and state management
HTML5 Canvas API Rendering the game elements (snake, food, obstacles, grid)
LocalStorage API Persisting high score between game sessions
Touch Events API Implementing touch controls for mobile devices
CSS Media Queries Creating a responsive design that adapts to different screen sizes

No external libraries or frameworks were used in order to:

  • Minimize loading time and application size
  • Avoid unnecessary dependencies
  • Demonstrate core web development skills
  • Maintain full control over all aspects of the game

5. HCI Principles Applied

HCI Principle Implementation in Snake Game
Visibility of System Status
  • Real-time score display and high score tracking
  • Clear "Game Over" state with final score
  • "GAME PAUSED" indicator when game is paused
  • Visual distinction between snake head and body
  • Pause button text changes between "Pause" and "Resume" to reflect current state
Match Between System and Real World
  • Intuitive controls matching directional expectations (arrow keys, WASD)
  • Swipe gestures on mobile that match the intended direction of movement
  • Snake movement and growth mechanics that match the real-world understanding of how a snake moves
  • Color coding matching real-world associations (green snake, red food, purple obstacles)
User Control and Freedom
  • Ability to pause and resume the game at any time
  • Clear "Play Again" option after game over
  • Difficulty selection to match player skill level
  • Multiple control schemes (arrow keys, WASD, touch) giving users choice
Consistency and Standards
  • Consistent visual design throughout the application
  • Standard game controls familiar to users (arrow keys)
  • Consistent button styling and interactive elements
  • Game mechanics consistent with typical snake game expectations
Error Prevention
  • Prevention of 180-degree turns (cannot immediately reverse direction)
  • Smart obstacle placement to prevent impossible game situations
  • Buffer zone around the snake's head when placing new obstacles
  • Prevention of food or obstacles spawning on occupied spaces
Recognition Rather Than Recall
  • On-screen instructions explaining controls
  • Visual differentiation between snake, food, and obstacles
  • Clear labels on all interactive elements
  • Keyboard controls visually represented with key symbols
Flexibility and Efficiency of Use
  • Multiple control schemes (keyboard, touch)
  • Difficulty options catering to different skill levels
  • Both button and keyboard shortcut for pausing
  • Responsive design working across various devices
Aesthetic and Minimalist Design
  • Clean, distraction-free game interface
  • Only relevant game information displayed
  • Visually pleasing color scheme
  • Clear visual hierarchy of information
Help Users Recognize, Diagnose, and Recover from Errors
  • Clear game over screen with score feedback
  • Obvious play again option
  • Visual feedback when collisions occur
Help and Documentation
  • Simple instructions provided directly on screen
  • Visual representations of keyboard controls
  • Self-explanatory UI elements
Accessibility
  • High contrast colors for visibility
  • Multiple control options (keyboard, touch)
  • Appropriate ARIA labels on interactive elements
  • Focus indicators for keyboard navigation
  • Responsive design supporting different screen sizes

8. Conclusion

The Snake game demonstrates the application of HCI principles to create an engaging, accessible, and user-friendly web application. By focusing on intuitive controls, clear feedback, error prevention, and responsive design, the game provides an enjoyable experience for users across different devices and skill levels.

Key achievements of this project include:

  • Implementing classic Snake gameplay with modern features and enhancements
  • Creating a fully responsive design that works on both desktop and mobile devices
  • Applying HCI principles systematically to improve user experience
  • Balancing challenge and accessibility through difficulty options
  • Developing the entire game as a single HTML file with vanilla JavaScript and no external dependencies

This project demonstrates how even a simple game can be enhanced through careful attention to HCI principles, resulting in a more engaging and accessible experience for users.
